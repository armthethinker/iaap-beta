Getting Started

The frontend is built with a series of tools.

  1. npm or the Node Package Manager helps Grunt run. You'll need to install NodeJS with a packaged installer. npm comes with Node, but you should update it via $ sudo npm install npm -g.
  2. Bower is our package manager for the browser. Install the command line interface with a $ npm install -g bower.
  3. Grunt is a task runner. Install the command line interface with $ sudo npm install -g grunt-cli.

Now that you have the command line setup, download the repository. Rename the project. Do a global search for

project-init-php

and replace it your project's name.

Next, run these tasks to get your environment situated and the files you need built.

$ npm install
$ bower update
$ grunt setup

All of your files should be set up and ready to go! Try $ grunt to start the persistent task.

If something goes wrong, first try deleting /node_modules/ and /bower_components, then rerun the three lines above.

Grunting

Grunt is a JavaScript task runner, meaning that it automates frontend tasks. We use it to compile various CSS and JS files, prefix our CSS so we can write cleanly without worrying about vendor prefixes, it cleans /dist/ so the most up-to-date files are being used, it minifies both our CSS and JS so download speeds are quicker, and it watches our code so it can reload our browser when we change it.

Full Grunt flow (not all tasks run everything):

  1. Clean /dist/
  2. Copy any files it needs from /bower_components/
  3. Compile all of the Less files into CSS
  4. Concatenate CSS, including files from /bower_components/
  5. Concatenate JS, including files from /bower_components/
  6. Replace all instances of 'glyphicon' in CSS with 'fa'
  7. Add and remove vendor prefixes in CSS
  8. Minify CSS
  9. Minify JS
  10. Compile the README.md to HTML
  11. Clean up any files that were only used in the build

When done, /dist/ is ready to go and the app's frontend is built.

$ grunt (slim)

This is what you'll use all the time.

Runs the base build tasks and watches for updates in your files. When you save, Grunt will rebuild the frontend. It's a little smart too and only runs tasks that are related to the file you've saved.

If you have the LiveReload Chrome extension, your browser will update when Grunt is done building.

$ grunt css (slim)

Runs the core CSS tasks, doesn't minify.

$ grunt js (slim)

Runs the core JS tasks, doesn't minify.

$ grunt full (production)

Runs all of the CSS, JS, and other tasks. Minifies. Packages for production. Run this before you push to the Git repo.

$ grunt deploy (production)

Deploys all code except for /img/, via SFTP, to the specified server.

You'll need to set up a .ftppass with your magic username and password in it. Rename .ftppass-template to .ftppass and follow the instructions. Don't worry, Git will ignore your password.

$ grunt cleanup (utility)

Cleans your folder tree of /bower_components/, /node_modules/, and /dist/.

$ grunt setup (utility)

This task does the inital build. It is mostly $ grunt full.

Grunt Tasks

  • Autoprefixer: vendor prefixes for CSS, adds and removes based on the browser variable.
  • Contrib-Clean: cleans files and folders.
  • Contrib-Concat: concatenate files.
  • Contrib-Copy: copy files and folders.
  • Contrib-CSSmin: compress CSS.
  • Contrib-LESS: compile LESS files to CSS.
  • Contrib-Uglify: minify files with UglifyJS.
  • Contrib-Watch: run tasks whenever watched files change.
  • md2html: converts Markdown files to HTML.
  • SFTP-Deploy: deploys code over SFTP.
  • Text-Replace: general purpose text replacement.

You can find the Grunt tasks in Gruntfile.js and package.json for more detail.

Bowering

Bower is a package manager. It pulls and manages project dependencies (libaries built in HTML, CSS, and JS) and keeps it separate from the rest of the app files. When Grunt compiles the frontend, it pulls files out of /bower_components/ and mixes them appropriately with our app files. When adding a dependency, do it via Bower either by searching for it or using the $ bower install API.

You will need to write some lines in Grunt to add the dependency to the core app files.

Components

We use a base of customized Bootstrap and build up with our own code. When using files, make sure to only pull from /static as this is where the final, compiled, minified files land.

HTML

  • Bootstrap: HTML, CSS, and JS framework.

CSS

  • Bootstrap: HTML, CSS, and JS framework.
  • Font Awesome: icon set.
  • Animate.css: CSS animations.
  • Select2: more robust select tool.
  • Anchor: builds inline anchors for our docs.
  • iCheck: better checkboxes and radios.

The CSS is often written in Less, a CSS preprocessor. It allows the use of variables and nesting (among other things). The files are broken into components, each file only holding that component's Less. There is a variable file, /css/less/variables.less, which holds the majority of the Less variables. Another file, /css/less/build.less, is the file that pulls together all of the Less components.

JS

  • Bootstrap: HTML, CSS, and JS framework.
  • FixieJS: adds filler text (DEV).
  • HolderJS: adds filler images (DEV).
  • Select2: more robust select tool.
  • Anchor: builds inline anchors for our docs.
  • ScrollTo: smooth in-page link scrolling.
  • iCheck: better checkboxes and radios.

PHP

  • UIFunk: PHP arrays for random data generation (DEV).
